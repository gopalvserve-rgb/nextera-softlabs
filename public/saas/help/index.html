01🚀 First login

From zero to your first lead in under five minutes.

  1. Open your workspace URL.

    Visit https://crm.smartcrmsolution.com/t/<your-slug>/ — replace <your-slug> with the workspace name you signed up with (eg. vserve). Bookmark it.

  2. Log in with the admin email.

    Use the email + password you set during signup. If you forgot the password, click "Forgot password?" on the login screen.

  3. Set your company branding.

    Settings → Company. Upload your logo, set the company name, and pick a brand colour. These show on the login screen, the sidebar, the topbar, and the Android app icon area.

  4. Add your team.

    People → Users → Add user. Enter their email, set a role, optionally set a daily/monthly lead cap. They'll get an invite email.

  5. Customise statuses for your pipeline.

    Settings → Statuses. Add the stages you actually use (eg. New, Contacted, Qualified, Quote Sent, Won, Lost) and pick colours for each.

✅
Done with the basics? Next decide where leads come from — webhook (most flexible), Meta Lead Ads, Google Ads, CSV import, or all of them.

02👥 Users & roles

Add team members and pick what they can see and do.

  1. Add a user.

    People → Users → Add user. Fill name, email, phone, role.

  2. Pick the right role.
    adminFull access. Can manage settings, billing, integrations.
    managerCan see all leads, assign to anyone, but can't edit settings.
    agentSees only leads assigned to them.

    Custom roles? Settings → Roles lets you build your own combinations.

  3. Set lead caps (optional).

    Each user can have a Daily lead cap and Monthly lead cap. The auto-assigner respects these — agents who hit their cap stop receiving new leads until midnight / month-rollover.

  4. Activate or deactivate.

    To pause a user, toggle their Active flag off. Their leads stay visible to managers; they can no longer log in.

03🟢 Statuses

The funnel stages every lead moves through.

  1. Open Settings → Statuses.
  2. Add a status. Click Add status, give it a name (eg. Site Visit Booked), pick a colour, and set a sort order. Lower numbers appear higher in the list.
  3. Mark final statuses. Tick Is final for terminal stages like Won, Lost, Junk. Final statuses are excluded from "Open leads" counts and dashboard funnels.
  4. Default status for new leads. The system uses New by default. If you delete or rename it, set the new default in Settings → Statuses → Default for new leads.
💡
Status colours are used everywhere — kanban, dashboard pie charts, lead list pills, even the WhatsApp chat header. Pick semantic colours (green = good, red = lost, yellow = waiting).

04🔍 Sources & tags

Track where leads come from (Source) and add free-form labels (Tags).

  1. Add sources. Settings → Sources. Examples: Website, Meta Ads, Google Ads, Walk-in, Referral, Trade Show 2026.
  2. Sources are auto-created from webhooks. When a lead arrives via /hook/website with source=Foobar, a Source row called Foobar is created automatically the first time it's seen.
  3. Tags are free-form. Type any label on a lead — VIP, callback-tomorrow, quote-sent-v2. Comma-separated. Good for ad-hoc segmentation.

05➕ Custom fields

Track anything specific to your business — budget, requirement, property type, course interested in, etc. Webhooks pick them up automatically.

Step-by-step

  1. Settings → Custom Fields → Add field.
  2. Fill the form.
    Keycampaign_name_new — internal identifier, lowercase, no spaces. This is what webhooks send.
    LabelCampaign Name — what users see on the lead form.
    TypeText / Number / Date / Dropdown / Multi-select
    Options(Dropdown only) one option per line, eg. Hot, Warm, Cold
    Show in listTick to make it a column on the lead grid
    RequiredTick to enforce on the lead form
  3. Save. The field is now live — visible on every lead's form, available as a webhook target, and listed in filter dropdowns.
crm.smartcrmsolution.com/t/<slug>/#/admin · Custom Fields
The Key value (left) is exactly what your webhook sender uses (right). Match them character-for-character.

Sending the value via webhook

NextEra Softlabs looks for the field in three places (in this order):

1. Top level (most common)

{
  "name": "Manu",
  "phone": "9971...",
  "campaign_name_new": "Nh-24 Villa"
}

2. cf_ prefix

{
  "name": "Manu",
  "phone": "9971...",
  "cf_campaign_name_new": "Nh-24 Villa"
}

Or nested under extra:

{
  "name": "Manu",
  "phone": "9971...",
  "extra": {
    "campaign_name_new": "Nh-24 Villa",
    "budget_band": "50L–1Cr"
  }
}
✅ Success looks like: open the new lead → scroll to the Custom Fields section → see Campaign Name = Nh-24 Villa. If Show in list is on, it also appears as a column in the leads grid.
⚠️ Watch out for:
  • Renaming a Key after data exists. Old rows still reference the old key — they'll appear blank under the new name. If you must rename, contact support to re-key.
  • Spaces or capitals in Key. Use budget_band not Budget Band or budgetBand. Webhooks send the literal key.
  • Field added but webhook still drops the value. Verify the field's Active flag is on (Settings → Custom Fields → toggle column).

06📦 Products

A catalogue of what you sell. Used by Quotations and on the lead form.

  1. Settings → Products → Add product. Name, description, price, active flag.
  2. Pick a product on a lead. Lead form → Product dropdown searches your catalogue.
  3. Use in Quotations. Quotations module pulls from this list — the product picker shows price, your team can override per-quote.

07⚖️ Auto-assign rules

When a lead arrives, send it to the right person automatically.

  1. Settings → Auto-assign Rules → Add rule.
  2. Define the match.
    FieldSource / City / Product / Phone area / Custom field (cf:<key>)
    Operatorequals / contains / starts with / ends with
    ValueThe literal value to match
    Assigned toOne or more users (round-robin among them)
    PriorityLower = higher priority. First matching rule wins.
  3. Round-robin within the rule. When multiple users are assigned to one rule, the system picks the user with the fewest open leads today — keeping load even.
  4. Lead cap fallback. If the chosen user has hit their daily/monthly cap, the rule skips to the next user.
  5. Disabling a rule. Untick Active on the rule row.
💡
For more sophisticated routing — quotas, percentages, conditional logic — use Campaigns instead. Auto-assign rules are the simple "if-this-then-that" path.

08🎯 Campaigns

Group leads by ad campaign or source, route them across agents using one of five modes.

  1. Settings → Campaigns → Add campaign.
  2. Pick a distribution mode.
    On DemandLeads stay unassigned until an agent calls Pull. Pulls happen in batches you size below.
    EqualNew leads spread evenly across all agents over time.
    Round RobinStrict cursor — A, then B, then C, then back to A. Order is deterministic across server restarts.
    PercentageEach agent gets a fixed % of new leads. Must add up to 100%.
    ConditionalRoutes by Source / Status / Custom-field rules you define inside the campaign.
  3. Choose lead-filter rules. Inside the campaign modal, decide which incoming leads belong to it — eg. source = Meta Ads AND cf:campaign_name_new contains "Nh-24". Leads matching join the campaign automatically.
  4. Add agents. Pick the users in this campaign. For Percentage mode, set each user's % directly.
  5. Removed-user action. When an agent leaves the campaign, decide what happens to their leads: reassign within campaign, unassign, or leave with the agent.
  6. Pause / resume. Toggle the Active switch. Paused campaigns don't accept new leads but keep existing assignments.

09⏱️ TAT (response SLA)

Alert when a lead sits untouched too long.

  1. Settings → TAT.
  2. Set the SLA. "First response within X minutes/hours of lead creation". Eg. 30 minutes for hot Meta Lead Ads.
  3. Set escalation. If the agent doesn't respond in time, optionally re-assign to a manager OR push a notification to the team.
  4. Watch the TAT report. Reports → TAT Report shows breaches by user, by stage, and by source. Excel download available.

10🚚 Project stages

For service / project businesses — track delivery progress separately from the sales pipeline.

  1. Settings → Project Stages → Add stage. Examples: Site Survey, Design, Procurement, Installation, Handover.
  2. Set expected days. The dashboard flags any project sitting in a stage longer than expected.
  3. Assign a default role per stage. Eg. Procurement stage auto-assigns to a Procurement Manager.
  4. Move leads through stages. Lead detail page → Project Stage dropdown.

11👥 Duplicate handling

What happens when the same phone or email comes in twice.

  1. Settings → Duplicates.
  2. Pick a policy.
    allowCreate both, but flag the new one as "duplicate of #X" (default).
    rejectRefuse the second submission entirely.
    assign_same_userCreate the lead, but assign it to whoever owns the original.
    skip_assignmentCreate unassigned (manager will triage).
  3. Set the window. "Treat as duplicate if same phone arrives within X hours" — default 24h.
  4. See duplicates. Lead list → Filter → "⚠️ Duplicates only" or use the bulk Dedupe button.

12🔌 Website API webhook

The most flexible way to push leads in — your website, your forms, any tool that can POST JSON.

  1. Get your API key. Settings → Website API → click Regenerate if you don't see one yet. Copy the key.
  2. POST a lead. Send a JSON body to:
    https://crm.smartcrmsolution.com/t/<your-slug>/hook/website
    With either header x-api-key: <your-key> or Authorization: Bearer <your-key>.
  3. Required fields.
    nameRequired Lead's name
    phoneRequired Mobile number (with country code)
    emailOptional
    sourceRecommended Eg. Website, Meta Ads
    productOptional
    notesOptional Free text
    tagsOptional Comma-separated string or JSON array
    city, state, country, pincode, company, addressOptional
    campaign_name / campaign_idOptional Aliases accepted: campaign_name_new, campaignName, campaign_title etc.
    Any custom field keyOptional The system reads every active custom field and writes matches into extra_json. Send the value at b[<key>], b[cf_<key>], or b.extra[<key>].
  4. Test it. 
    curl -X POST https://crm.smartcrmsolution.com/t/<slug>/hook/website \
  -H "Content-Type: application/json" \
  -H "x-api-key: <your-key>" \
  -d '{
    "name": "Test Lead",
    "phone": "9876543210",
    "email": "test@example.com",
    "source": "Website",
    "product": "Demo",
    "notes": "Came from contact form"
  }'
    You should get {"ok":true,"id":123}.
  5. Watch live activity. Settings → Website API → Recent inbound shows the last 50 webhook hits with raw body and any errors. Use this to debug bad mappings.
⚠️
Phones with fewer than 10 digits are auto-flagged as Junk instead of being rejected, so they're visible without polluting the active pipeline.

  1. Set up the trigger.

  2. Add an "Action — API by Pabbly" step.

    Open Leads. The test lead appears at the top, status New. Click it — the custom-field tab should show Campaign Name = Nh-24 Villa, Campaign Id = 120243….

    Pabbly only fires when the workflow is enabled. Confirm the green toggle in the top right of the Pabbly editor.

✅ Success looks like: the lead appears in Leads within 2–3 seconds of the test fire. Settings → Website API → Recent inbound shows a row with processed = 1 and the body keys you mapped.
⚠️ Common mistakes (and how to spot them):
  • Lead arrives but a field is empty. You dragged the labelled chip in Pabbly (the one that shows "1. Res1 Campaign Name : Nh-24 Villa") instead of the bare value chip. Re-drag using the inner value tag.
  • 401 Invalid API key. Either the key is wrong, or the header name has a typo. Header MUST be x-api-key (lowercase, hyphen). Settings → Website API → Recent inbound shows a redacted prefix of whatever Pabbly actually sent.
  • Lead created but custom field missing. The custom field doesn't exist in NextEra Softlabs yet. Add it at Settings → Custom Fields with the same Key Pabbly is sending — eg. campaign_name_new.
  • "campaign_Name" vs "campaign_name". Keys are case-sensitive. NextEra Softlabs accepts a generous list of aliases for the campaign-name field, but custom-field keys must match exactly.
💡
The Recent inbound table at NextEra Softlabs → Settings → Website API logs the last 50 hits with status, redacted key prefix, and body keys. Best debug tool when something looks wrong.

14🧩 Make.com

Same idea as Pabbly — Make.com (formerly Integromat) sends leads via the Website API.

  1. Build a scenario. Trigger from Facebook / Typeform / a Sheet / wherever.
  2. Add an "HTTP — Make a request" module.
  3. Method: POST. URL: the same /t/<slug>/hook/website.
  4. Headers: x-api-key: <your-key>, Content-Type: application/json.
  5. Body type: Raw → JSON (application/json). Build the JSON payload using the field mapper.
  6. Run once with a sample input to verify, then turn the scenario on.
💡
NextEra Softlabs has a dedicated Make.com card on Settings → Integrations — it shows a sample payload and a Test-send button so you can verify the connection without leaving the CRM.

15⚡ Zapier & n8n

Same recipe again — POST JSON to the Website API webhook.

  1. Zapier: Add a "Webhooks by Zapier — POST" action. URL = /t/<slug>/hook/website. Add headers + JSON body. Map the fields. Done.
  2. n8n: Use the "HTTP Request" node. Method POST, URL same, Authentication = Header Auth (x-api-key), Body JSON, map fields.
  3. For high volume: Use Pabbly's per-execution pricing for predictable cost. Make/n8n charge by operations, Zapier by tasks.

16📘 Meta (Facebook) Lead Ads

Two ways to wire it up — direct (Facebook → NextEra Softlabs) or via Pabbly. Direct is faster but has less flexibility.

Option A — Direct via Facebook OAuth (recommended for one or two pages)

  1. Settings → Facebook → Connect Facebook account. A Facebook OAuth popup opens.
  2. Approve permissions. NextEra Softlabs needs leads_retrieval + pages_manage_metadata.
  3. Pick the Page(s). Tick the pages whose leads you want to receive.
  4. Pick the Form(s). Tick the lead-gen forms inside each page.
  5. Webhook auto-subscribes. NextEra Softlabs registers our central webhook with Meta automatically — no need to copy URLs into Meta's Developer console.
  6. Test it. Use Meta's "Lead Ads Testing" tool inside Facebook Business Suite to fire a test lead. It should appear in NextEra Softlabs within a few seconds.

Option B — Pabbly bridge (when direct doesn't work, eg. agency-managed pages)

Use the Pabbly Connect recipe above with Facebook Lead Ads as trigger.

17🟡 Google Ads ValueTrack

Capture {campaignid}, {keyword}, {gclid} and friends straight off the landing page.

  1. On your landing page, add ValueTrack params to the form action URL or read them off window.location.search: 
    ?campaign={campaignid}&campaign_name={campaignname}&network={network}&keyword={keyword}&gclid={gclid}&adgroupid={adgroupid}&matchtype={matchtype}&device={device}
  2. Submit them with the lead. Include the params in your POST body to /hook/website (alongside name/phone/email).
  3. What gets stored. NextEra Softlabs normalises Google Ads params into both first-class columns AND meta_json — so any reports already built on either keep working:
    gclidFirst-class column. Use it to upload offline conversions back to Google Ads.
    gad_campaignidFirst-class column.
    utm_*UTM source/medium/campaign/term/content all populated.
    SourceAuto-set to Google Ads when gclid or campaign is present (unless you pass source explicitly).
    TagsCampaign name + network are auto-added as tags.
  4. Reports. Reports → Group by → UTM campaign shows leads + conversions per Google Ads campaign.

18📥 CSV import (LeadSquared, Zoho, HubSpot, Salesforce)

Migrating from another CRM? The Import wizard knows the column names.

  1. Export from your old CRM. CSV format. UTF-8 encoded. Make sure phone numbers aren't in scientific notation (Excel does this — open in Google Sheets or use plain Numbers/text format).
  2. Settings → Integrations → Import → Upload CSV.
  3. Pick your source platform. LeadSquared / Zoho / HubSpot / Salesforce / Generic. The wizard auto-detects column names for the first four.
  4. Map any unmapped columns. The wizard shows a side-by-side: their column → NextEra Softlabs field. Custom fields are pre-listed so you can map directly into them.
  5. Preview the first 5 rows. Make sure phones look right, statuses match (the wizard maps them to your existing statuses or asks you to create new ones), no NaN or shifted columns.
  6. Import. The job runs in the background. You'll see a counter as rows land. Duplicates respect your duplicate policy.

19📅 Calendly

Auto-create a lead (or update an existing one) when someone books a meeting.

  1. Get your per-user webhook token. Settings → Users → click your name → copy the Calendly webhook token.
  2. In Calendly, set up a webhook. Account → Integrations → Webhooks → Create. URL:
    https://crm.smartcrmsolution.com/hook/calendly/<your-token>
    Subscribe to invitee.created.
  3. Match logic. When a Calendly booking arrives, NextEra Softlabs matches by phone and email. If a lead exists, it updates next_followup_at to the booking time. If not, it creates a new lead assigned to you.

20💬 WhatsApp Cloud API — connect in 5 minutes

From zero to sending your first WhatsApp message in 5 minutes — no Meta dev console, no copy-pasting tokens.

ℹ️
Before you start: you need (a) a Facebook account that owns your business, (b) a phone number you can receive an OTP on, and (c) admin access in NextEra Softlabs. That's it.

Step-by-step (with screenshots)

  1. Open Settings → WhatsApp

    From the left sidebar in NextEra Softlabs, click ⚙️ Settings, then open the WhatsApp tab.

    Step 1 — Sidebar → Settings → WhatsApp tab
    • Click the gear icon at the bottom of the left sidebar.
    • In the Settings rail, click WhatsApp.
    • The right pane shows the WhatsApp settings card.
  2. Click "Connect with Facebook"

    The big blue button on the WhatsApp settings card opens Meta's embedded sign-in flow in a popup.

    Step 2 — The big blue button on the WhatsApp settings card
    • Make sure popups are allowed on this site (check the URL bar).
    • If you want to keep using the WhatsApp Business app on your phone at the same time, tick the Coexistence mode checkbox first (see section 21 for trade-offs).
    • Click the blue Connect with Facebook button.
  3. Authorise in the Facebook popup

    A Facebook window opens. Sign in with the FB account that owns your business pages.

    Step 3 — Facebook authorisation popup (one-time)
    • Sign in with your business Facebook account.
    • Review the permissions Meta lists — they're all needed.
    • Click the blue Continue button at the bottom right of the popup.
    If the popup says "URL blocked", it means a different redirect-URI was used. Check that Settings → WhatsApp shows the same domain as your CRM and try again.
  4. Pick or create the WhatsApp Business Account

    Meta asks which WhatsApp Business Account (WABA) to link. If you have one from earlier campaigns, pick it. Otherwise click Create new WABA and give it your business name.

    • WABA name → your business as you want customers to see it.
    • Timezone → matches your support hours (used for delivery analytics).
    • Currency → INR if you're billing customers in rupees.
  5. Pick or add a phone number

    Meta asks which phone number to use. You can pick an existing landline / business mobile, or let Meta provision a brand-new number.

    Step 4 — Choose an existing number, or add new (Meta sends OTP)
    • If using an existing number, you'll get an OTP — voice call or SMS. Pick whichever can reach that number.
    • The number must not currently be on the regular WhatsApp / WhatsApp Business app — unless you ticked Coexistence mode in step 2.
    • Enter the OTP, click Verify, then Continue.
    Stuck on OTP? Try the other option (voice instead of SMS, or vice versa). Indian carriers sometimes block international SMS from Meta.
  6. NextEra Softlabs finishes the wiring automatically

    The popup closes and NextEra Softlabs auto-registers the webhook with Meta and saves the access token. No manual URL copying.

    Done — NextEra Softlabs auto-registers the webhook with Meta
    • You'll see a green check + your phone number + "webhook subscribed".
    • NextEra Softlabs also fetches any existing message templates you'd previously created in Meta.
  7. Send a test message

    Open the Chat tab on the left sidebar. Type a phone number and a quick "Hello from NextEra Softlabs 👋" and send. Within seconds the message lands on that phone.

    • Use a number you can check on a real phone (yours or a colleague's).
    • If the test message is the first contact, you have to use a pre-approved template — see section 23.
    • If the test message is a reply to a recent inbound (24-hour customer service window), free-text works.

Path B — Manual setup (only if Embedded Sign-In doesn't work)

Use this only if the embedded flow fails for you — e.g. when your FB account doesn't have business-asset access on the page you need.

  1. In Meta Developer Console — create a WhatsApp Business app. Grab Phone Number ID, WhatsApp Business Account ID, and a permanent Access Token (System User token, never expires).
  2. Set the webhook URL to https://crm.smartcrmsolution.com/t/<your-slug>/hook/whatsapp_webhook. Subscribe to messages.
  3. Set the verify token to any string. Paste the same value into NextEra Softlabs → Settings → WhatsApp → Verify token.
  4. Paste the access token + phone number ID into NextEra Softlabs Settings → WhatsApp.
  5. Save and verify. Send yourself a test message.

21🤝 WhatsApp Coexistence mode

Keep using the WhatsApp Business app on your phone and the NextEra Softlabs web inbox on the same number — at the same time.

When to use it

  • Your team already replies on the WhatsApp Business app and you don't want to migrate everyone to web.
  • You want push notifications to keep landing on the phone.
  • You're okay with the limitations (read on).

How to enable

  1. Start the embedded sign-in flow from Settings → WhatsApp (see section 20).
  2. Tick "Coexistence mode" in the NextEra Softlabs popup BEFORE clicking the Facebook button.
    • If you forget, you'll have to disconnect and re-run the flow.
  3. Approve in Facebook. Meta shows a different consent screen for coexistence — read the differences carefully.
  4. Both surfaces are live. Phone reply → shows in NextEra Softlabs. NextEra Softlabs reply → pops up on the phone. Push notifications keep working on the phone as before.
⚠️
Trade-offs vs full Cloud API: no template messages outside the 24-hour customer service window; no broadcast lists; group messaging is API-read-only (you can read group messages but not send); you can no longer block users from the app side. You can switch off Coexistence later if your needs grow — but that requires re-registering the number.

22📱 Multiple WhatsApp numbers

Connect more than one WhatsApp number — eg. one for sales, one for support — and route inbound by line.

  1. Open Settings → WhatsApp → Connected numbers

    You'll see your current number listed with a green check.

    Add a second number — same embedded sign-in flow
  2. Click "Connect another number"

    Triggers the same embedded sign-in flow as your first number. Pick a different phone number (or have Meta provision one).

    • You can use the same WABA (recommended for billing) or a different one.
    • Each connected number gets its own templates, webhook, and (optional) AI Bot config.
  3. Set the default sender per agent (optional)

    Settings → Users → click an agent → Default WhatsApp sender. Whatever you pick is the default for that user's outbound messages.

    • Sales reps → set default to your sales number.
    • Support reps → set default to your support number.
    • Leave blank → NextEra Softlabs picks the number the conversation arrived on.
  4. Send-from picker on new messages

    When you compose a brand-new outbound (no existing thread), the chat box shows a dropdown — pick which connected number to send from.

  5. Per-number filter on the inbox

    The chat thread list has a per-phone filter at the top of the sidebar. Click it to show only conversations for one of your numbers.

23📨 WhatsApp templates

Pre-approved messages you can send outside the 24-hour customer service window. Required for cold outreach, broadcasts, and reminders.

  1. Settings → WhatsApp → Templates → Create template
    Create template form — Meta usually approves in 0–24h

34🔥 Hot-lead Heat Detection

Detect when a lead is signalling buying intent and ping the right salesperson within seconds — automatically.

ℹ️
How it works: Every inbound WhatsApp message, call summary, and chat reply is scored against your tenant-configurable keyword list. When the score crosses a threshold, the lead is tagged Hot, a heat alert popup fires on the assigned salesperson's screen, and a push notification lands on their phone.

Set up in 3 steps

  1. Open Settings → Heat Detection

    From the sidebar, Settings → Heat Detection. You'll see the default keyword list and the threshold slider.

    Heat Detection settings — keywords + weights + threshold
  2. Tune the keywords

    Defaults work for most B2B sales — add 3–5 keywords specific to your business and adjust weights.

    • Positive keywords (positive weight) — words that signal buying intent: "pricing", "demo", "schedule", "order", "purchase", "ready", "interested", "GST", "invoice"
    • Negative keywords (negative weight) — words that signal disinterest: "not interested", "too expensive", "remove", "stop"
    • Threshold — 7 is a good starting point. Lower = more alerts (noisier). Higher = fewer alerts (quieter).
  3. Pick notification channels

    Choose how the assigned salesperson gets alerted. Multi-select.

    • In-app popup — modal pops up in the CRM with the lead name + the message that triggered the alert (recommended)
    • Mobile push notification — uses your existing FCM setup (section 28)
    • WhatsApp to manager — sends a templated alert to a fallback recipient (set in the same panel)
    • Email — uses your SMTP setup (section 27)

Test it

There's a 🧪 Test button on the settings page. It picks a recent lead and simulates an inbound matching your keyword list, then fires the full alert pipeline. If the popup + push notification arrive within 5 seconds, you're set.

⚠️
Heat ≠ AI Bot. Heat detection runs after the AI Bot has replied — both fire on the same inbound. The AI Bot answers the customer; heat detection wakes up the salesperson.
